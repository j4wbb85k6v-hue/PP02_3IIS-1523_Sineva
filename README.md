## 🔧 Аппаратная часть (Задание №1)

### Схема подключения I2C
| Компонент | Arduino Uno (Master) | Arduino Nano (Slave) |
| :--- | :--- | :--- |
| **Питание** | 5V | VIN |
| **Земля** | GND | GND |
| **Шина I2C** | A4 (SDA) | A4 (SDA) |
| **Шина I2C** | A5 (SCL) | A5 (SCL) |
| **Вход** | A0 (Потенциометр) | — |
| **Выход** | — | D13 (Светодиод) |

### Логика работы
- **Master (Uno):** Считывает аналоговый сигнал с потенциометра (0-1023). Преобразует его в байт (0-255) и отправляет по I2C на адрес `0x08`.
- **Slave (Nano):** Принимает байт. Если значение **> 127** (что соответствует ~512 в исходном сигнале), зажигает встроенный светодиод на плате (Pin 13).

### 📂 Файлы прошивок
- `master_uno_potentiometer.ino` — Код для Arduino Uno.
- `slave_nano_led.ino` — Код для Arduino Nano.

## 🌐 Сетевая часть (Задание №2)

### Конфигурация сервера (Windows Server / Linux)
Виртуальная машина настроена как контроллер домена / отдельный сервер со следующими ролями:

#### 1. DHCP Сервер
- **Подсеть:** 192.168.10.0/24
- **Пул адресов:** 192.168.10.50 — 192.168.10.200
- **Шлюз по умолчанию:** 192.168.10.1
- **DNS Сервер:** 192.168.10.2

#### 2. DNS Сервер
- **Зона прямого просмотра:** `local`
- **A-запись:** `testclient.local` → `192.168.10.101`
